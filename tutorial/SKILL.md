# Tutorial - 编程教程编写专家

## 工作流程

编写编程教程时，按以下步骤执行：

### 1. 内容调研

**目标**：全面了解题目涉及的技术，确定教程深度和广度。

**执行步骤**：
1. 使用 `plan` 技能规划调研方向
2. 使用 `web_search` 搜索：
   - 官方文档链接
   - 权威教程资源
   - 最新语法特性
   - 常见使用场景
   - 已知问题和踩坑点
3. 整理调研结果，列出：
   - 核心语法点（估算数量）
   - 主要使用场景
   - 常见问题和坑点

### 2. 确定文档结构

根据内容量决定使用单文档还是多文档：

| 评估维度 | 单文档 (<5000字) | 多文档 (>5000字) |
|---------|-----------------|-----------------|
| 语法点数量 | < 10个 | >= 10个 |
| 使用场景 | < 5个 | >= 5个 |
| 子主题独立性 | 低，相互依赖 | 高，可独立学习 |
| 学习曲线 | 平缓 | 有明显进阶路径 |

**决策流程**：
```
调研完成
    ↓
估算总字数和语法点数量
    ↓
内容较少？ ──是──→ 使用 [单文档模板](references/tutorial-template.md)
    ↓ 否
内容较多？ ──是──→ 参考 [多文档拆分指南](references/multi-doc-guide.md)
```

### 3. 编写教程

#### 单文档模式

直接参考 [tutorial-template.md](references/tutorial-template.md) 的完整结构编写。

#### 多文档模式

1. 参考 [multi-doc-guide.md](references/multi-doc-guide.md) 选择合适的拆分方式
2. 创建文档大纲，确定每个文档的主题
3. 按顺序编写各个文档

### 4. 质量检查

完成编写后，检查以下内容：

- [ ] 所有代码示例可直接运行
- [ ] 代码有必要的注释
- [ ] 复杂概念有图示或类比说明
- [ ] 每个语法点都有示例
- [ ] 踩坑点有明确的解决方案
- [ ] 文档间引用正确（多文档模式）

## 文档结构说明

### 第1部分：概述

**目标**：让读者快速了解这项技术是什么、为什么要学。

**内容要点**：
- **是什么**：技术定义、核心功能、主要特点
- **为什么**：解决的问题、相比替代方案的优势、适用人群

### 第2部分：基本语法

**目标**：全面且详细地讲解所有常用语法。

**写作规范**：
- 每个语法点必须包含：语法说明、参数解释、代码示例
- 参数说明使用表格形式
- 示例代码从简单到复杂递进
- 关键代码行添加注释

**示例格式**：
```python
# 不好的示例
result = calculate(x, y)

# 好的示例
# 使用 calculate() 函数计算两个数的和
# x: 第一个加数
# y: 第二个加数
result = calculate(x=10, y=20)
print(result)  # 输出: 30
```

### 第3部分：使用场景

**目标**：通过具体场景帮助读者理解何时使用这项技术。

**每个场景的组成**：
1. 场景描述：什么情况下会遇到这个问题
2. 解决思路：为什么选择这个技术
3. 完整代码：可直接运行的示例
4. 输出结果：展示运行效果

### 第4部分：不适用的场景

**目标**：帮助读者避免误用，了解技术边界。

**写作方式**：
- 列出明确不适用的场景
- 说明为什么不适用
- 给出替代方案建议

### 第5部分：最佳实践

**目标**：传授经验和规范，帮助读者写出更好的代码。

**常见维度**：
- 命名规范和代码风格
- 性能优化技巧
- 安全注意事项
- 可维护性建议

### 第6部分：常见踩坑点

**目标**：提前预警，节省读者排错时间。

**每个坑点的组成**：
1. **现象**：会出现什么错误/异常/不符合预期的行为
2. **原因**：为什么会这样（深入解释）
3. **解决方案**：正确的写法或规避方法
4. **预防措施**：如何避免再次遇到

### 第7部分：总结

**目标**：帮助读者回顾和规划下一步学习。

**内容要点**：
- 核心要点回顾（3-5条）
- 学习路径建议
- 进阶方向指引
- 推荐资源

## 代码示例规范

1. **可运行**：所有代码示例应该可以直接复制粘贴运行
2. **完整性**：包含必要的导入语句和初始化代码
3. **渐进性**：从最简单示例开始，逐步增加复杂度
4. **注释**：关键行添加注释说明
5. **输出展示**：展示代码运行的预期输出
