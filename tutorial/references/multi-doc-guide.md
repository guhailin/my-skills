# 多文档拆分指南

当教程内容较多时，按主题拆分成多个文档。遵循以下原则：

## 拆分原则

1. **单一职责**：每个文档只讲一个核心主题
2. **循序渐进**：从基础到进阶，文档间有依赖关系
3. **独立完整**：每个文档可以独立阅读，但会引用前置内容

## 推荐拆分方式

### 方式一：按难度拆分
适合技术体系完整、学习曲线明显的主题

```
tutorial/
├── 01-快速入门.md       # 概述 + 最基础用法
├── 02-核心概念.md       # 基础语法详解
├── 03-进阶技巧.md       # 高级用法
├── 04-实战案例.md       # 完整项目示例
└── 05-常见问题.md       # 踩坑点 + 最佳实践
```

### 方式二：按功能模块拆分
适合功能丰富、各模块相对独立的技术

```
tutorial/
├── 01-概述与安装.md
├── 02-模块A详解.md
├── 03-模块B详解.md
├── 04-模块C详解.md
├── 05-综合应用.md
└── 06-常见问题.md
```

### 方式三：按使用场景拆分
适合应用场景多样化、不同场景差异大的技术

```
tutorial/
├── 01-基础入门.md
├── 02-场景A实战.md
├── 03-场景B实战.md
├── 04-场景C实战.md
└── 05-总结与进阶.md
```

## 文档命名规范

1. **序号前缀**：使用 `01-`、`02-` 等两位数序号，方便排序
2. **简短主题**：用2-4个词概括文档核心内容
3. **使用连字符**：单词间用 `-` 连接
4. **文件扩展名**：统一使用 `.md`

示例：
- `01-快速入门.md`
- `02-核心语法.md`
- `03-异步编程.md`

## 文档间引用方式

在文档开头说明前置依赖：

```markdown
# 文档标题

> **前置知识**：本文档假设你已了解 [基础语法](../02-基础语法.md)

> **阅读时长**：约 30 分钟
```

在文档结尾给出下一步指引：

```markdown
## 下一步

学习完本文档后，你可以：
- 继续学习 [进阶技巧](./03-进阶技巧.md)
- 查看 [实战案例](../examples/)
```

## 每个文档的结构

即使是拆分后的文档，每个文档内部仍遵循单文档模板的基本结构：

```markdown
# 1. 概述
简要介绍本文档主题

# 2. 核心内容
详细讲解

# 3. 示例代码
足够的代码示例

# 4. 常见问题
本文档特有的踩坑点

# 5. 小结
本阶段要点回顾
```

## 内容量判断标准

### 适合单文档（< 5000 字）
- 语法点少于 10 个
- 使用场景少于 5 个
- 踩坑点少于 5 个

### 适合多文档（> 5000 字）
- 语法点多且需要详细展开
- 多个相对独立的子主题
- 需要循序渐进的学习路径
